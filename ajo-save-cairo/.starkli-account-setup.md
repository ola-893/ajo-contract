## Alternative Setup Methods

### Option 1: Use sncast account (Recommended for Scarb projects)

Since you're using Scarb, `sncast` is better integrated. Here's how to set it up:

1. **Import account to sncast:**
```bash
sncast account import --name deployer \
  --address 0x0634f84Cd94953222B6E170E0ED7610AF8b191B1130Fa937AB5F4c476c01C539 \
  --private-key <YOUR_PRIVATE_KEY> \
  --type argent \
  --add-profile deployer
```

2. **Or import from keystore file:**
```bash
sncast account import --name deployer \
  --address 0x0634f84Cd94953222B6E170E0ED7610AF8b191B1130Fa937AB5F4c476c01C539 \
  --keystore ~/.starkli-wallets/deployer/keystore.json \
  --type argent
```

Note: The `class_hash` above is for ArgentX v0.3.0. If you're using a different version or Braavos, you'll need the correct class hash.

## Recommended Next Steps

1. **Get your private key from ArgentX/Braavos wallet**
   - Open the browser extension
   - Go to Settings → Export Private Key
   - Copy the private key

2. **Use sncast to import the account:**
```bash
sncast account import --name deployer \
  --address 0x0634f84Cd94953222B6E170E0ED7610AF8b191B1130Fa937AB5F4c476c01C539 \
  --private-key <YOUR_PRIVATE_KEY> \
  --type argent \
  --add-profile deployer
```

3. **Verify the account:**
```bash
sncast account list
```

4. **Set environment variables:**
```bash
export STARKNET_ACCOUNT=deployer
export STARKNET_RPC=https://starknet-sepolia.public.blastapi.io/rpc/v0_7
```

## For Deployment

Once your account is set up with sncast, you can deploy using:

```bash
# Declare contracts
sncast --account deployer declare --contract-name AjoFactory

# Deploy
sncast --account deployer deploy --class-hash <CLASS_HASH>
```

This approach is cleaner for Scarb-based projects and doesn't require starkli for deployment.
